# 一. Unix、linux系统排查

## 1. 当前登录用户及定时任务

确定当前登录用户只需执行w(who)命令。命令w 将显示登录用户的用户ID、从什么系统登录，以及当前在本系统中运行什么。使用w命令开始和结束每次响应，可以识别在目标系统中执行操作的时间范围，以及收集证据时谁在本系统上。

## 2. 获取运行的进程

应急响应过程中取得所有运行的进程的快照非常重要。可以通过使用标准的ps命令实现。不同的Unix版本，所得到的输出略有区别。

在Solaris系统中使用ps –eaf，而在FreeBSD和Linux系统中则要使用ps –aux。

命令ps的输出中一个最重要的字段是START字段，该字段显示了进程的开始时间，对于查出攻击发生的时间非常有帮助。

## 3. 获取开放端口和监听应用程序

Netstat命令是查看Unix系统中开放端口的第一选择。 在Linux下，netstat命令具有一个-p选项，将应用程序的名字和进程ID映射到开放端口。

lsof命令列举所有运行进程及其所打开的文件描述符。为使lsof只列举打开网络套接字的进程，可以使用：

*lsof -i*

## 4. 获取所有文件的创建，修改和访问时间

使用ls命令获取文件系统中所有文件的创建，修改和访问时间。下面几行展示了如何获得时间/日期戳并将输出保存到usb盘上：

访问时间 *ls* *–alRu > /mnt/usb/access*

修改时间 *ls* *–alRc > /mnt/usb/modification*

创建时间 *ls* *–alR > /mnt/usb/creation*

获得时间/日期戳（需要很长的时间），便可以更加自由的检查文件，例如使用find命令获取在某特定时间范围内访问、修改或创建的全部文件的列表。

## 5. 获取系统日志

大多数UNIX的日志在/var/log和/var/adm目录下，各种UNIX派生系统日志的具体位置有所不同。在此之前，有必要了解针对特定系统的日志存贮位置。

比较重要的二进制日志文件：

l utmp,用w工具访问；

l wtmp,用last工具访问；

l lastlog,用lastlog工具访问；

l 进程记账日志,用astcomm工具访问

常见的ASCII文本日志文件：

l apache日志--/var/log/httpd/access_log；

l ftp日志—xferlog；

l 命令历史记录文件；

l /var/log/messages;

检查/etc/syslog.conf以及其他守护进程的配置文件以确定其余日志的位置。

## 6. 获取重要的配置文件

Unix保存一些特定配置文件，攻击者常常会访问或修改这些文件。要检查各个配置文件以查找后门位置、未授权的信任关系和未授权的用户ID，这点很重要。下面列出所需获取的文件：

l /etc/passwd，查找未授权的用户帐号或权限

l /etc/shadow，确保每个帐号都要求密码认证

l /etc/groups，查找权限的升级和访问范围的扩大

l /etc/hosts 列出本地DNS条目

l /etc/hosts.equiv 检查信任关系

l ~/.rhosts，检查基于用户的信任关系

l /etc/hosts.allow和/etc/hosts.deny(TCP Wrapper规则)

l /etc/syslog.conf，确定日志文件的位置

l /etc/rc，检查启动文件

l Cronttab文件，列出计划事件

l /etc/inetd.conf，列出监听的服务

1.6  文件系统信息

| <br> | <br> |
| --- | --- |
| **dir /t:a /a /s /o:d c:\** | **显示C****盘上所有访问时间的递归式目录清单** |
| **dir /t:w /a /s /o:d c:\** | **显示C****盘上所有修改时间的递归式目录清单** |
| **dir /t:c /a /s /o:d c:\** | **显示C****盘上所有创建时间的递归式目录清单** |
